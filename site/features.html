Feature Overview

A brief map of the core capabilities that Heimdall brings to every pull request, helping maintainers review faster with consistent, explainable automation.

Webhook Triage

Runs on pull request open, edit, sync, reopen, ready-for-review, and relevant label events to keep triage current.

Smart Labels

Applies and maintains triage:low-effort, triage:ai-slop, and a single size label from size/XS through size/XL.

Explainable Scoring

Publishes one managed triage comment with a transparent score breakdown and updates it on every rerun.

Duplicate Detection

Checks newly opened pull requests for likely duplicates using patch and metadata similarity to reduce repeated work.

Human Override

Honors the reviewed-by-human label so maintainers can bypass automation on pull requests already validated manually.

Trusted Bypasses

Skips triage for trusted automation accounts and trusted title patterns, keeping noise low and signal high.
